= Extract message properties

In your mapping, you can extract values from the message properties, and then write those values to a database table.

For example, this mapping extracts the `header` property `f4` from the message properties, and writes it to the `col3` column in the specified table:

[source,yaml]
----
topic:
  topic_name:
    keyspace_name:
      table_name:
        mapping: 'col1=key.f1, col2=value.f1, __ttl=value.f2, __timestamp=value.f3, col3=header.f4'
----

Use one or more individual mapping statements to extract the desired values and write them to table columns.
